Tags That Relate To Text and Content

  • h1 ... h6 Heading levels. These create hierarchy within your page. This is for the font size from from h1 to h6. h1 is always used for the title. h6 is the smallest writting text)
  • p tag is used to write a paragraph
  • Strong Strong Importance, bold by default. This is an inline element, used inside of paragraphs and headings to put more importance on same of the text.
  • em Emphasis italic by default. This is an inline element, usedinside of a paragraphs and headings to put emphasis on certain words within the text.
  • a Anchor Used is to create links - think of it as anchoring to another location. This is an inline element, and can be used inside of paragraphs and headings.
  • ul Unordered list.
  • ol, Ordered list.
  • li List Item, used inside of ul and ol elements. ol or ul tag must be written inside the li (list item)
  • span Similar to strong and em but with no default styling and no semantic meaning. you would use CSS to style it how you want.
  • img src="" alt="" An image, must have an alt attribute, which describes the image. Is written just as img in style css

    • The only tags you need to know!


  • Header Denotes a heading within the document. Often used for the logo and navigation area on a page, but can also be used within other elements (such as an article), to denote the heading for that section of content.
  • Main The main content of your page (only one per page).
  • Section A section of content.
  • Footer Denotes a footer within a document. Lie header, it is often as the primary footer for an entire page, but you can have footers within other elements as well.
  • Nav Used for major navigational elements (not all links, or lists of links must be in a nav). If you have multiple nav elements, they should contain context for their purpose, with either an aria-label or aria-labelled-by.
  • Div A division (or box) - no semantic meaning. These are used to organize your content, generally so you can style layouts with CSS.
  • Article A piece of content on your page can stand on it's own.

The box model: most elements are block level elements by default.
They have a width of 100% of their parent, a height of 0. the height grows to match the content.
